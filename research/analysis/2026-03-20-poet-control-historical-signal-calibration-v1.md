# POET Control Historical Signal Calibration v1

Date: March 20, 2026

## Purpose

Apply the calibration framework to a control case:

- `POET` as a standalone narrowing target

This control is useful because `POET` is not noise.

It has:

- recurring mentions
- real adjacency to photonics
- real discussion of customers, qualifications, and architecture

But the question is not whether `POET` is a real company or a real adjacent
expression.

The question is:

- should it have created its own standalone structural-pressure candidate from
  the broad public flow?

The answer for this control case is:

- no

## Facts

1. By `2026-01-21`, `POET` had recurring archive mentions.

2. Those mentions usually framed `POET` as:
   - weaker than `AAOI`
   - not yet hyperscaler-qualified
   - a workaround path rather than the current chokepoint
   - more retail-crowded than underfollowed

3. The case therefore has enough signal to be tempting, but not enough to
   justify creating a new standalone bounded universe.

## Assumptions

1. The right control question is not whether `POET` belonged inside the
   photonics universe.

2. The right control question is whether `POET` should have created a separate
   narrowing decision on its own.

3. For this case, a failure to narrow is the correct outcome.

## Calibration Judgment

### What This Case Says

1. Repeated mentions alone are not enough.

2. Real adjacency to a hot universe is not enough.

3. Even a case with customer, qualification, and architecture language can
   still correctly fail narrowing if it lacks:
   - standalone demand pressure
   - stress rationale
   - bottleneck logic

4. This is the right kind of control because it looks superficially interesting
   but should still fail.

### Threshold Implication

1. `system_grounding` alone is not sufficient.

2. `visible_beneficiary` adjacency alone is not sufficient.

3. A standalone structural-pressure candidate should require actual
   demand-pressure evidence, not just branch speculation inside an existing
   universe.

### Caveats

1. This control evaluates `POET` as a standalone narrowing target, not as a
   secondary object inside photonics.

2. The case is sourced primarily from the reconstructed archive rather than a
   richer broad-web control build.

3. More controls will still be useful, especially outside photonics-adjacent
   narratives.
