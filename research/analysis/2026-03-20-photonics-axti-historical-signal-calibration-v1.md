# Photonics -> AXTI Historical Signal Calibration v1

Date: March 20, 2026

## Purpose

Apply the two-pass calibration framework to the first benchmark case:

- `Photonics -> AXTI`

This is the first worked example.

It is based on:

- the reconstructed AleaBito archive
- the existing replay notes
- pre-cutoff public non-X evidence already collected in the repo

It is not yet a fully automated measurement pipeline.

   based on archive-visible, upstream-neutral clues rather than full broad-web
   crawling.

3. The oracle pass may still undercount total relevant signal, so the oracle
   metrics below should be treated as lower bounds rather than final ceiling
   estimates.

## Calibration Judgment

### What This Case Says

1. A valid narrowing decision did not require full chokepoint proof.

2. The blind pass only surfaced about half of the oracle-independent signals, yet still justified narrowing.

3. Role coverage mattered more than raw signal count:
   - demand pressure
   - system grounding
   - visible beneficiaries
   - stress hint
   - one upstream clue
   were enough

4. The main value of the oracle pass was not to justify narrowing, but to show
   how much stronger the case already was in principle than the blind pass alone
   could see.

### Threshold Implication

1. Promotion should not require bottleneck verification-level evidence.

2. Promotion should likely require:
   - demand pressure
   - system grounding
   - visible beneficiary coverage
   - at least one stress hint

3. A blind pass can succeed even with modest recall if it captures the right
   role coverage early enough.

### Caveats

1. The blind pass here is based on one reconstructed archive source, so source
   diversity is artificially low.

2. The oracle pass is still a lower bound because it includes only the
   pre-cutoff public evidence already reconstructed in the repo.

3. This is one case only, so it should not be used to set hard production
   thresholds alone.
